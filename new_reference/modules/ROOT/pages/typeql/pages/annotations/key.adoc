= `@key` annotation
:page-aliases: {page-version}@new_reference::typeql/pages/statements/key.adoc

The `@key` annotation is used
// tag::overview[]
to specify key attributes for entities and relations.
// end::overview[]

== Syntax

.Syntax
[,typeql]
----
<type label> owns <attribute type label> @key;
----

The `@key` annotation does not accept any arguments.

== Usage

The `@key` annotation is used to mark an owned attribute as a key, which in fact declares two constraints:

- The xref:{page-version}@new_reference::typeql/pages/annotations/unique.adoc[unique] constraint as an equivalent of `@unique`.
- The xref:{page-version}@new_reference::typeql/pages/annotations/card.adoc[cardinality] constraint as an equivalent of `@card(1)`.

For example, the following statement declares that any `content` should have exactly one key `id` with a unique (across all `content` s) value:

=== Subtyping
Refer to xref:{page-version}@new_reference::typeql/pages/annotations/unique.adoc#subtyping[unique constraint subtyping] and xref:{page-version}@new_reference::typeql/pages/annotations/card.adoc#subtyping[card constraint subtyping] for the respective constraints' subtyping behavior.

== Limitations

Cannot be used with the xref:{page-version}@new_reference::typeql/pages/annotations/unique.adoc[`@unique` annotation] the xref:{page-version}@new_reference::typeql/pages/annotations/card.adoc[`@card` annotation] as it produces same constraints.

See xref:{page-version}@new_reference::typeql/pages/annotations/unique.adoc#_value_types[`unique` annotation supported value types] for the list of supported value types.
